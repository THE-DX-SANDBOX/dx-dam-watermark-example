# Support

## Getting Help

Use the repository documentation first:

- `README.md` for the repo overview
- `docs/` for architecture, deployment, and registration guidance
- `START_HERE.md` and `docs/first-clone-setup.md` for onboarding

## Where to Ask Questions

- Open a GitHub issue for bugs, gaps in documentation, or reproducible problems
- Open a GitHub issue for enhancement requests or proposed improvements

When asking for help, include:

- what you were trying to do
- the command or workflow you ran
- the expected result
- the actual result
- environment details such as OS, Node version, and cluster context when relevant

## What Support Does Not Cover

This repository does not guarantee:

- private support channels
- production operational support
- SLAs or response-time guarantees

## Security Issues

For security-sensitive problems, do not open a public issue. Follow `SECURITY.md` instead.
